# دليل تشغيل مشروع Durataljoud (الواجهة الخلفية - Backend) على الخادم (Server)

هذا الملف يوضح الخطوات اللازمة لتشغيل المشروع على أي خادم، مع مراعاة التعديل على ملف البيئة الخاص بالمشروع.

## 1. إعداد متغيرات البيئة (.env)
هذه خطوة هامة جدًا. يجب التأكد من تعديل ملف `.env` ليتوافق مع إعدادات الخادم والمنفذ وقاعدة البيانات الجديدة.

ملف `.env` يجب أن يحتوي على المتغيرات التالية:
```env
DB_PATH ="mongodb+srv://..."
PORT = "3003"
NODE_MODE = "production"
JWT_SECRET_KEY= "durataljoud"
EXPIRE_DATE = 30d
```

### 🔴 التعديلات التي يجب مراعاتها:
1. **`PORT` (المنفذ):** 
   - قم بتغيير الـ `3003` إلى رقم المنفذ (Port) المتاح والذي تريد تشغيل المشروع عليه على الخادم.
2. **`DB_PATH` (قاعدة البيانات):**
   - قم بتحديث رابط الاتصال (Connection String) ليتصل بقاعدة البيانات الخاصة ببيئة الإنتاج (سواء كانت MongoDB محلية أو سحابية Atlas).
   - تأكد من اسم قاعدة البيانات (Database Name) والبيانات السرية الخاصة بها.
3. **`NODE_MODE`:**
   - تأكد من تغيير القيمة إلى `"production"` عند التشغيل الفعلي على الخادم لضمان أفضل أداء وإغلاق أخطاء التطوير.

## 2. تشغيل المشروع على الخادم (Run the Project)
يُفضل تشغيل المشروع باستخدام أداة `pm2` لكي يبقى يعمل في الخلفية حتى بعد إغلاق سطر الأوامر، بالإضافة إلى إعادة تشغيله تلقائياً في حال حدوث خطأ.

للتشغيل لأول مرة:
```bash
pm2 start dura.js --name "durataljoud_backend"
```

أوامر هامة في `pm2`:
- **لعرض حالة المشروع:** `pm2 status`
- **لعرض السجلات والأخطاء (Logs):** `pm2 logs durataljoud_backend`
- **لإعادة تشغيل المشروع (Restart):** `pm2 restart durataljoud_backend`
- **لإيقاف المشروع:** `pm2 stop durataljoud_backend`
